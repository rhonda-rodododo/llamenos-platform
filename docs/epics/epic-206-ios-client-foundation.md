# Epic 206: iOS Client Foundation

## Goal

Build the native SwiftUI iOS app with crypto integration via UniFFI, secure key storage via iOS Keychain, and the core auth flow (login, onboarding, PIN). This validates the full stack: Rust crypto → Swift bindings → SwiftUI UI.

## Context

The React Native mobile app (llamenos-mobile) hit fundamental testing issues with Detox + Fabric — `typeText` on controlled TextInputs doesn't trigger `onChangeText` on the last field in a group, and `FabricUIManagerIdlingResources` never becomes idle. Native iOS avoids these issues entirely: XCUITest interacts with real UIKit/SwiftUI controls, no bridge layer.

### Architecture Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| UI Framework | SwiftUI | Modern, declarative, matches React mental model |
| Min iOS | 17.0 | `@Observable` macro, simplified navigation, modern APIs |
| Dependency Mgmt | Swift Package Manager | Apple-native, no CocoaPods complexity |
| Architecture | MVVM + `@Observable` | SwiftUI-native, minimal boilerplate |
| Crypto | UniFFI from `packages/crypto/` | Same Rust code as desktop — one auditable implementation |
| Key Storage | iOS Keychain | Hardware-backed on Secure Enclave devices |
| Networking | URLSession | Built-in, no third-party deps needed |
| VoIP | Twilio Voice iOS SDK | Proven, supports CallKit integration |

### What llamenos-mobile Already Proved

The React Native prototype validated:
- UniFFI binding generation works (XCFramework with arm64 + arm64-sim)
- Expo Modules can wrap UniFFI calls (sync functions, hex string boundary)
- CryptoProvider abstraction (native → JS fallback) works well
- PIN pad (custom button grid) avoids keyboard issues entirely
- VoIP background modes (`UIBackgroundModes: voip, audio, fetch, remote-notification`)

The native app takes these learnings and implements them directly in Swift.

## CryptoService Design

The most critical component. The nsec (private key) MUST stay inside `CryptoService` — views and view models only see the pubkey.

```swift
@Observable
final class CryptoService {
    private(set) var pubkey: String?
    private(set) var npub: String?
    private var nsecHex: String?  // NEVER exposed outside this class

    // Key generation
    func generateKeypair() -> (nsec: String, npub: String) {
        let kp = LlamenosCore.generateKeypair()
        self.nsecHex = kp.secretKeyHex
        self.pubkey = kp.publicKey
        self.npub = kp.npub
        return (kp.nsec, kp.npub)
    }

    // Import existing key
    func importNsec(_ nsec: String) throws {
        let kp = try LlamenosCore.keypairFromNsec(nsec)
        self.nsecHex = kp.secretKeyHex
        self.pubkey = kp.publicKey
        self.npub = kp.npub
    }

    // PIN-encrypt the nsec for storage
    func encryptForStorage(pin: String) throws -> EncryptedKeyData {
        guard let nsec = nsecHex else { throw CryptoError.noKeyLoaded }
        return try LlamenosCore.encryptWithPin(nsec: nsec, pin: pin, pubkeyHex: pubkey!)
    }

    // Decrypt nsec from storage
    func decryptFromStorage(_ data: EncryptedKeyData, pin: String) throws {
        let nsec = try LlamenosCore.decryptWithPin(data: data, pin: pin)
        try importNsec(nsec)
    }

    // Create auth token (nsec never leaves)
    func createAuthToken(method: String, path: String) throws -> AuthToken {
        guard let nsec = nsecHex else { throw CryptoError.noKeyLoaded }
        let timestamp = Int(Date().timeIntervalSince1970 * 1000)
        return try LlamenosCore.createAuthToken(
            secretKeyHex: nsec,
            timestamp: timestamp,
            method: method,
            path: path
        )
    }

    // Encrypt note — encryptNoteForRecipients uses pubkey-only ECIES wrapping
    // (ephemeral keypairs wrap the note key for each recipient's pubkey).
    // The author's nsec is NOT needed for wrapping — only for later decryption.
    // If note signing is added in the future, nsecHex would be passed here.
    func encryptNote(payload: String, adminPubkeys: [String]) throws -> EncryptedNote {
        guard let pubkey else { throw CryptoError.noKeyLoaded }
        return try LlamenosCore.encryptNoteForRecipients(
            payloadJson: payload,
            authorPubkey: pubkey,
            adminPubkeys: adminPubkeys
        )
    }

    // Lock: clear nsec from memory. pubkey/npub remain for display ("locked as npub1...")
    func lock() {
        nsecHex = nil
        // NOTE: pubkey and npub are intentionally NOT cleared — they're needed to
        // show which identity is locked and to match incoming envelopes for display.
    }

    // Check if unlocked
    var isUnlocked: Bool { nsecHex != nil }
}
```

### Memory Safety

- `nsecHex` is `private` — no external access
- `lock()` sets it to `nil` — Swift ARC deallocates the string
- Called on `scenePhase == .background` after configurable timeout
- Called on `UIApplication.willTerminateNotification`

## Keychain Integration

```swift
final class KeychainService {
    private let service = "org.llamenos.hotline"

    func store(key: String, data: Data, biometric: Bool = false) throws {
        var query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: key,
            kSecValueData as String: data,
            kSecAttrAccessible as String: kSecAttrAccessibleWhenUnlockedThisDeviceOnly,
        ]

        if biometric {
            let access = SecAccessControlCreateWithFlags(
                nil,
                kSecAttrAccessibleWhenUnlockedThisDeviceOnly,
                .biometryCurrentSet,
                nil
            )
            query[kSecAttrAccessControl as String] = access
        }

        let status = SecItemAdd(query as CFDictionary, nil)
        guard status == errSecSuccess || status == errSecDuplicateItem else {
            throw KeychainError.storeFailed(status)
        }

        if status == errSecDuplicateItem {
            let updateQuery: [String: Any] = [
                kSecClass as String: kSecClassGenericPassword,
                kSecAttrService as String: service,
                kSecAttrAccount as String: key,
            ]
            let updateStatus = SecItemUpdate(updateQuery as CFDictionary, [kSecValueData as String: data] as CFDictionary)
            guard updateStatus == errSecSuccess else {
                throw KeychainError.updateFailed(updateStatus)
            }
        }
    }

    func retrieve(key: String) throws -> Data? {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: key,
            kSecReturnData as String: true,
            kSecMatchLimit as String: kSecMatchLimitOne,
        ]

        var result: AnyObject?
        let status = SecItemCopyMatching(query as CFDictionary, &result)

        if status == errSecItemNotFound { return nil }
        guard status == errSecSuccess else { throw KeychainError.retrieveFailed(status) }
        return result as? Data
    }

    func delete(key: String) {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: key,
        ]
        SecItemDelete(query as CFDictionary)
    }
}
```

Key storage layout:
- `"encrypted-keys"` — PIN-encrypted nsec (JSON: salt, ciphertext, nonce)
- `"hub-url"` — Server endpoint URL
- `"device-id"` — Unique device identifier

## PIN Pad Design

Custom SwiftUI view — no TextField, no keyboard. Avoids all the issues Detox had with TextInput.

```swift
struct PINPadView: View {
    @Binding var pin: String
    let maxLength: Int  // 4 or 6
    let onComplete: (String) -> Void

    private let buttons = [
        ["1", "2", "3"],
        ["4", "5", "6"],
        ["7", "8", "9"],
        ["", "0", "backspace"],  // Use SF Symbol "delete.left" for rendering
    ]

    var body: some View {
        VStack(spacing: 20) {
            // PIN dots
            HStack(spacing: 12) {
                ForEach(0..<maxLength, id: \.self) { i in
                    Circle()
                        .fill(i < pin.count ? Color.primary : Color.clear)
                        .stroke(Color.secondary, lineWidth: 1)
                        .frame(width: 16, height: 16)
                }
            }

            // Number grid
            ForEach(buttons, id: \.self) { row in
                HStack(spacing: 24) {
                    ForEach(row, id: \.self) { button in
                        PINButton(label: button) {
                            handleTap(button)
                        }
                    }
                }
            }
        }
    }

    private func handleTap(_ button: String) {
        if button == "⌫" {
            if !pin.isEmpty { pin.removeLast() }
        } else if !button.isEmpty && pin.count < maxLength {
            pin.append(button)
            if pin.count == maxLength {
                onComplete(pin)
            }
        }
    }
}
```

## API Service

```swift
final class APIService {
    private let baseURL: URL
    private let cryptoService: CryptoService
    private let session: URLSession

    init(baseURL: URL, cryptoService: CryptoService) {
        self.baseURL = baseURL
        self.cryptoService = cryptoService
        self.session = URLSession(configuration: .default)
    }

    func request<T: Decodable>(
        method: String,
        path: String,
        body: (any Encodable)? = nil
    ) async throws -> T {
        var request = URLRequest(url: baseURL.appendingPathComponent(path))
        request.httpMethod = method

        // Schnorr auth token
        let token = try cryptoService.createAuthToken(method: method, path: path)
        let authHeader = """
        {"pubkey":"\(token.pubkey)","timestamp":\(token.timestamp),"token":"\(token.token)"}
        """
        request.setValue("Bearer \(authHeader)", forHTTPHeaderField: "Authorization")

        if let body {
            request.setValue("application/json", forHTTPHeaderField: "Content-Type")
            request.httpBody = try JSONEncoder().encode(body)
        }

        let (data, response) = try await session.data(for: request)
        guard let httpResponse = response as? HTTPURLResponse,
              (200...299).contains(httpResponse.statusCode) else {
            throw APIError.requestFailed(response)
        }

        return try JSONDecoder().decode(T.self, from: data)
    }
}
```

## App Lifecycle & Security

```swift
@main
struct LlamenosApp: App {
    @Environment(\.scenePhase) private var scenePhase
    @State private var appState = AppState()
    @State private var backgroundTimestamp: Date?

    private let lockTimeout: TimeInterval = 300  // 5 minutes

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environment(appState)
                .onChange(of: scenePhase) { oldPhase, newPhase in
                    switch newPhase {
                    case .background:
                        backgroundTimestamp = Date()
                    case .active:
                        if let bg = backgroundTimestamp,
                           Date().timeIntervalSince(bg) > lockTimeout {
                            appState.cryptoService.lock()
                            appState.isLocked = true
                        }
                        backgroundTimestamp = nil
                    default:
                        break
                    }
                }
        }
    }
}
```

## Testing Strategy

### Unit Tests (XCTest)

```swift
class CryptoServiceTests: XCTestCase {
    func testGenerateKeypair() {
        let service = CryptoService()
        let (nsec, npub) = service.generateKeypair()
        XCTAssertTrue(nsec.hasPrefix("nsec1"))
        XCTAssertTrue(npub.hasPrefix("npub1"))
        XCTAssertNotNil(service.pubkey)
        XCTAssert(service.isUnlocked)
    }

    func testLockClearsNsecButKeepsPubkey() {
        let service = CryptoService()
        _ = service.generateKeypair()
        let originalPubkey = service.pubkey!
        service.lock()
        XCTAssertFalse(service.isUnlocked)
        XCTAssertEqual(service.pubkey, originalPubkey)  // pubkey stays for display
        XCTAssertNotNil(service.npub)  // npub stays for display
    }

    func testPINEncryptDecrypt() throws {
        let service = CryptoService()
        _ = service.generateKeypair()
        let originalPubkey = service.pubkey!

        let encrypted = try service.encryptForStorage(pin: "1234")
        service.lock()

        try service.decryptFromStorage(encrypted, pin: "1234")
        XCTAssertEqual(service.pubkey, originalPubkey)
    }
}
```

## Risk Assessment

- **Low risk**: SwiftUI + `@Observable` — well-documented, stable APIs on iOS 17+
- **Medium risk**: UniFFI XCFramework linking — proven in llamenos-mobile but native Xcode project is different from Expo
- **Low risk**: Keychain — standard Apple security API, well-documented
- **Medium risk**: XCUITest reliability — generally solid, but simulator-specific issues possible
- **Mitigation**: Start with CryptoService unit tests before UI tests; validate UniFFI linking first
