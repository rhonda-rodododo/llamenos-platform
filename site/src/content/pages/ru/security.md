## Если ваш хостинг-провайдер получает повестку

| Они МОГУТ предоставить | Они НЕ МОГУТ предоставить |
|------------------|---------------------|
| Метаданные звонков/сообщений (время, продолжительность) | Содержимое заметок, транскрипты, тела отчётов |
| Зашифрованные блобы базы данных | Имена волонтёров (сквозное шифрование) |
| Каких волонтёров и когда активировали | Записи каталога контактов (сквозное шифрование) |
| | Содержимое сообщений (зашифровано при поступлении, хранится как шифртекст) |
| | Ключи дешифрования (защищены вашим PIN, вашим провайдером идентификации и, опционально, аппаратным ключом безопасности) |
| | Ключи шифрования для каждой заметки (эфемерные — уничтожаются после оборачивания) |
| | Ваш HMAC-секрет для реверсии хешей телефонов |

**Сервер хранит данные, которые не может прочитать.** Метаданные (когда, как долго, какие аккаунты) видны. Содержимое (что было сказано, что было написано, кто ваши контакты) — нет.

## По функциям

Ваша уязвимость конфиденциальности зависит от того, какие каналы вы включаете:

### Голосовые звонки

| Если вы используете... | Третьи стороны могут получить доступ | Сервер может получить доступ | Контент E2EE |
|---------------|-------------------------|-------------------|--------------|
| Twilio/SignalWire/Vonage/Plivo | Аудио звонка (в реальном времени), записи звонков | Метаданные звонков | Заметки, транскрипты |
| Самостоятельный Asterisk | Ничего (вы контролируете) | Метаданные звонков | Заметки, транскрипты |
| Браузер-браузер (WebRTC) | Ничего | Метаданные звонков | Заметки, транскрипты |

**Повестка провайдера телефонии**: они имеют детальные записи звонков (время, номера телефонов, продолжительность). У них НЕТ заметок звонков или транскриптов. Запись по умолчанию отключена.

**Транскрипция**: Транскрипция выполняется полностью в вашем браузере с использованием ИИ на устройстве. **Аудио никогда не покидает ваше устройство.** Хранится только зашифрованный транскрипт.

### Текстовые сообщения

| Канал | Доступ провайдера | Хранение на сервере | Примечания |
|---------|-----------------|----------------|-------|
| SMS | Ваш провайдер телефонии читает все сообщения | **Зашифровано** | Провайдер сохраняет оригинальные сообщения |
| WhatsApp | Meta читает все сообщения | **Зашифровано** | Провайдер сохраняет оригинальные сообщения |
| Signal | Сеть Signal E2EE, но мост расшифровывает при поступлении | **Зашифровано** | Лучше SMS, не нулевые знания |

**Сообщения шифруются в момент поступления на ваш сервер.** Сервер хранит только шифртекст. Ваш провайдер телефонии или сообщений может по-прежнему иметь оригинальное сообщение — это ограничение тех платформ, а не то, что мы можем изменить.

**Повестка провайдера сообщений**: Провайдер SMS имеет полное содержимое сообщений. Meta имеет содержимое WhatsApp. Сообщения Signal E2EE до моста, но мост (работающий на вашем сервере) расшифровывает перед повторным шифрованием для хранения. Во всех случаях **ваш сервер содержит только шифртекст** — хостинг-провайдер не может прочитать содержимое сообщений.

### Заметки, транскрипты и отчёты

Весь контент, написанный волонтёрами, зашифрован сквозным шифрованием:

- Каждая заметка использует **уникальный случайный ключ** (прямая секретность — компрометация одной заметки не компрометирует другие)
- Ключи оборачиваются отдельно для волонтёра и каждого администратора
- Сервер хранит только шифртекст
- Дешифрование происходит в браузере
- **Пользовательские поля, содержимое отчётов и файловые вложения шифруются индивидуально**

**Изъятие устройства**: Без вашего PIN **и** доступа к аккаунту провайдера идентификации атакующие получают зашифрованный блоб, который вычислительно невозможно расшифровать. Если вы также используете аппаратный ключ безопасности, **три независимых фактора** защищают ваши данные.

## Конфиденциальность номеров телефонов волонтёров

Когда волонтёры получают звонки на личные телефоны, их номера открыты для провайдера телефонии.

| Сценарий | Номер телефона виден для |
|----------|------------------------|
| PSTN-звонок на телефон волонтёра | Провайдер телефонии, телефонный оператор |
| Браузер-браузер (WebRTC) | Никто (аудио остаётся в браузере) |
| Самостоятельный Asterisk + SIP-телефон | Только ваш Asterisk-сервер |

**Для защиты номеров телефонов волонтёров**: используйте браузерные звонки (WebRTC) или обеспечьте SIP-телефоны, подключённые к самостоятельному Asterisk.

## Недавно выпущено

Эти улучшения доступны уже сегодня:

| Функция | Преимущество для конфиденциальности |
|---------|-----------------|
| Зашифрованное хранение сообщений | SMS, WhatsApp и Signal сообщения хранятся как шифртекст на вашем сервере |
| Транскрипция на устройстве | Аудио никогда не покидает ваш браузер — обрабатывается полностью на вашем устройстве |
| Многофакторная защита ключей | Ваши ключи шифрования защищены PIN, провайдером идентификации и, опционально, аппаратным ключом безопасности |
| Аппаратные ключи безопасности | Физические ключи добавляют третий фактор, который невозможно скомпрометировать удалённо |
| Воспроизводимые сборки | Проверяйте соответствие развёрнутого кода открытому исходному коду |
| Зашифрованный каталог контактов | Записи контактов, связи и заметки зашифрованы сквозным шифрованием |
