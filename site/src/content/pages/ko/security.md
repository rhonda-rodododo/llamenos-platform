## 호스팅 제공업체가 소환장을 받는 경우

| 제공할 수 있는 것 | 제공할 수 없는 것 |
|-------------------|-------------------|
| 통화/메시지 메타데이터 (시간, 기간) | 메모 내용, 변환 기록, 신고 본문 |
| 암호화된 데이터베이스 블롭 | 복호화 키 (기기에 저장됨) |
| 언제 어떤 자원봉사자가 활동했는지 | 개별 메모 암호화 키 (임시) |
| SMS/WhatsApp 메시지 내용 | 전화번호 해시를 되돌리기 위한 HMAC 시크릿 |

**서버는 읽을 수 없는 데이터를 저장합니다.** 메타데이터 (언제, 얼마나, 누가)는 보입니다. 내용 (무엇을 말했는지, 무엇을 썼는지)은 보이지 않습니다.

## 기능별

개인정보 노출 정도는 활성화한 채널에 따라 달라집니다:

### 음성 통화

| 사용하는 것... | 제3자가 접근 가능 | 서버가 접근 가능 | E2EE 콘텐츠 |
|----------------|-------------------|------------------|-------------|
| Twilio/SignalWire/Vonage/Plivo | 통화 오디오 (실시간), 통화 기록 | 통화 메타데이터 | 메모, 변환 기록 |
| 자체 호스팅 Asterisk | 없음 (사용자가 제어) | 통화 메타데이터 | 메모, 변환 기록 |
| 브라우저 간 (WebRTC) | 없음 | 통화 메타데이터 | 메모, 변환 기록 |

**전화 서비스 제공업체 소환장**: 통화 세부 기록 (시간, 전화번호, 기간)을 가지고 있습니다. 통화 메모나 변환 기록은 가지고 있지 않습니다. 녹음은 기본적으로 비활성화되어 있습니다.

**변환 윈도우**: 약 30초의 변환 중에 오디오는 Cloudflare Workers AI에 의해 처리됩니다. 변환 후에는 암호화된 텍스트만 저장됩니다.

### 텍스트 메시징

| 채널 | 제공업체 접근 | 서버 저장 | 참고 |
|------|--------------|-----------|------|
| SMS | 전화 서비스 제공업체가 모든 메시지를 읽음 | 평문 | SMS의 고유한 제한 |
| WhatsApp | Meta가 모든 메시지를 읽음 | 평문 | WhatsApp Business API 요구 사항 |
| Signal | Signal 네트워크는 E2EE이지만, signal-cli 브리지가 복호화 | 평문 | SMS보다 나으나, 영지식 아님 |

**메시징 제공업체 소환장**: SMS 제공업체는 전체 메시지 내용을 가지고 있습니다. Meta는 WhatsApp 내용을 가지고 있습니다. Signal 메시지는 브리지까지 E2EE이지만, 브리지(사용자 서버에서 실행)는 평문을 가지고 있습니다.

**향후 개선**: 서버가 암호문만 저장하는 E2EE 메시지 저장소를 탐색하고 있습니다. [계획된 사항](#계획된-사항)을 참조하세요.

### 메모, 변환 기록 및 신고서

자원봉사자가 작성한 모든 콘텐츠는 종단 간 암호화됩니다:

- 각 메모는 고유한 랜덤 키를 사용합니다 (전방 비밀성)
- 키는 자원봉사자와 관리자를 위해 별도로 래핑됩니다
- 서버는 암호문만 저장합니다
- 복호화는 브라우저에서 발생합니다

**기기 압수**: PIN 없이 공격자는 암호화된 블롭을 얻습니다. 6자리 PIN과 600K PBKDF2 반복은 GPU 하드웨어에서 무차별 대입에 수시간이 걸립니다.

## 자원봉사자 전화번호 개인정보 보호

자원봉사자가 개인 전화로 통화를 받으면, 전화번호가 전화 서비스 제공업체에 노출됩니다.

| 시나리오 | 전화번호 노출 대상 |
|----------|-------------------|
| 자원봉사자 전화로 PSTN 통화 | 전화 서비스 제공업체, 통신사 |
| 브라우저 간 (WebRTC) | 없음 (오디오가 브라우저에 머무름) |
| 자체 호스팅 Asterisk + SIP 전화 | 사용자의 Asterisk 서버만 |

**자원봉사자 전화번호를 보호하려면**: 브라우저 기반 통화(WebRTC)를 사용하거나 자체 호스팅 Asterisk에 연결된 SIP 전화를 제공하세요.

**향후 개선**: 개인 전화번호를 노출하지 않고 통화를 받을 수 있는 네이티브 데스크톱 및 모바일 앱.

## 계획된 사항

신뢰 요구 사항을 줄이기 위한 개선 작업을 진행하고 있습니다:

| 기능 | 상태 | 개인정보 보호 이점 |
|------|------|-------------------|
| E2EE 메시지 저장소 | 계획됨 | SMS/WhatsApp/Signal이 암호문으로 저장 |
| 클라이언트측 음성 변환 | 계획됨 | 오디오가 브라우저를 떠나지 않음 |
| 네이티브 통화 수신 앱 | 계획됨 | 개인 전화번호가 노출되지 않음 |
| 재현 가능한 빌드 | 계획됨 | 배포된 코드가 소스와 일치하는지 확인 |
| 자체 호스팅 Signal 브리지 | 사용 가능 | 자체 인프라에서 signal-cli 실행 |

| 신고서 | 예 (E2EE) | 아니오 | 암호문만 |

| 통화 메타데이터 | 아니오 | 예 | 예 |
