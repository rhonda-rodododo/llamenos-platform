## 멀티 제공업체 전화 서비스

**5개 음성 제공업체** — Twilio, SignalWire, Vonage, Plivo 또는 자체 호스팅 Asterisk 중 선택하세요. 관리자 설정 UI 또는 설정 마법사에서 제공업체를 설정하세요. 코드 변경 없이 언제든지 제공업체를 전환할 수 있습니다.

**WebRTC 브라우저 통화** — 자원봉사자는 전화 없이 브라우저에서 직접 통화를 받을 수 있습니다. Twilio, SignalWire, Vonage 및 Plivo용 제공업체별 WebRTC 토큰 생성. 자원봉사자별 통화 선호도 설정 가능 (전화, 브라우저 또는 둘 다).

## 통화 라우팅

**동시 호출** — 발신자가 전화를 걸면, 근무 중이고 통화 가능한 모든 자원봉사자에게 동시에 벨이 울립니다. 첫 번째로 받는 자원봉사자가 통화를 가져가고, 나머지 벨은 즉시 중단됩니다.

**근무 기반 스케줄링** — 특정 요일과 시간대로 반복 근무를 만드세요. 자원봉사자를 근무에 배정하세요. 시스템이 자동으로 근무 중인 사람에게 통화를 라우팅합니다.

**대기 음악 큐** — 모든 자원봉사자가 통화 중이면 발신자는 설정 가능한 대기 음악이 있는 큐에 들어갑니다. 큐 시간 초과는 조절 가능합니다 (30-300초). 아무도 응답하지 않으면 음성 사서함으로 넘어갑니다.

**음성 사서함 대체** — 자원봉사자가 응답하지 않으면 발신자가 음성 사서함(최대 5분)을 남길 수 있습니다. 음성 사서함은 Whisper AI로 변환되고 관리자 검토를 위해 암호화됩니다.

## 암호화된 메모

**종단 간 암호화 메모** — 자원봉사자는 통화 중과 통화 후에 메모를 작성합니다. 메모는 브라우저를 떠나기 전에 ECIES (secp256k1 + XChaCha20-Poly1305)로 클라이언트 측에서 암호화됩니다. 서버는 암호문만 저장합니다.

**이중 암호화** — 모든 메모는 두 번 암호화됩니다: 작성한 자원봉사자용 한 번, 관리자용 한 번. 둘 다 독립적으로 복호화할 수 있습니다. 다른 사람은 내용을 읽을 수 없습니다.

**사용자 정의 필드** — 관리자가 메모의 사용자 정의 필드를 정의합니다: 텍스트, 숫자, 선택, 체크박스, 텍스트 영역. 필드는 메모 내용과 함께 암호화됩니다.

**초안 자동 저장** — 메모가 브라우저에서 암호화된 초안으로 자동 저장됩니다. 페이지가 새로고침되거나 자원봉사자가 다른 곳으로 이동해도 작업이 보존됩니다. 초안은 로그아웃 시 삭제됩니다.

## AI 음성 변환

**Whisper 기반 음성 변환** — 통화 녹음은 Cloudflare Workers AI의 Whisper 모델을 사용하여 변환됩니다. 변환은 서버 측에서 수행된 후 저장 전에 암호화됩니다.

**토글 제어** — 관리자가 전역적으로 음성 변환을 활성화/비활성화할 수 있습니다. 자원봉사자는 개별적으로 선택 해제할 수 있습니다. 두 토글은 독립적입니다.

**암호화된 변환 기록** — 변환 기록은 메모와 동일한 ECIES 암호화를 사용합니다. 저장되는 것은 암호문뿐입니다.

## 스팸 방지

**음성 CAPTCHA** — 선택적 음성 봇 감지: 발신자가 무작위 4자리 숫자를 듣고 키패드에 입력해야 합니다. 실제 발신자에게는 접근 가능하면서 자동 다이얼링을 차단합니다.

**요청 제한** — 전화번호별 슬라이딩 윈도우 요청 제한, Durable Object 스토리지에 유지. Worker 재시작 후에도 유지됩니다. 임계값 설정 가능.

**실시간 차단 목록** — 관리자가 개별 입력 또는 일괄 가져오기로 전화번호 차단 목록을 관리합니다. 차단은 즉시 적용됩니다. 차단된 발신자는 거부 메시지를 듣습니다.

**사용자 정의 IVR 안내** — 지원되는 각 언어에 대해 사용자 정의 음성 안내를 녹음하세요. 시스템은 IVR 흐름에 녹음을 사용하고, 녹음이 없으면 텍스트 음성 변환으로 대체합니다.

## 멀티 채널 메시징

**SMS** — Twilio, SignalWire, Vonage 또는 Plivo를 통한 수신 및 발신 SMS 메시징. 설정 가능한 환영 메시지로 자동 응답. 메시지는 스레드형 대화 뷰로 흐릅니다.

**WhatsApp Business** — Meta Cloud API (Graph API v21.0)를 통해 연결. 24시간 메시징 윈도우 내에서 대화를 시작하기 위한 템플릿 메시지 지원. 이미지, 문서 및 오디오에 대한 미디어 메시지 지원.

**Signal** — 자체 호스팅 signal-cli-rest-api 브리지를 통한 개인정보 보호 중심 메시징. 우아한 성능 저하를 통한 건강 모니터링. Workers AI Whisper를 통한 음성 메시지 변환.

**스레드형 대화** — 모든 메시징 채널이 통합 대화 뷰로 흐릅니다. 타임스탬프와 방향 표시기가 있는 메시지 버블. WebSocket을 통한 실시간 업데이트.

## 암호화된 신고

**신고자 역할** — 제보나 신고를 제출하는 사람을 위한 전용 역할. 신고자는 신고서와 도움말만 있는 간소화된 인터페이스를 봅니다. 역할 선택기가 있는 자원봉사자와 동일한 흐름으로 초대됩니다.

**암호화된 제출** — 신고 본문 내용은 브라우저를 떠나기 전에 ECIES로 암호화됩니다. 분류를 위한 평문 제목, 개인정보 보호를 위한 암호화된 내용. 첨부 파일은 별도로 암호화됩니다.

**신고 워크플로우** — 신고를 정리하기 위한 카테고리. 상태 추적 (열림, 접수됨, 해결됨). 관리자가 신고를 접수하고 스레드형 암호화 답변으로 응답할 수 있습니다.

## 관리자 대시보드

**설정 마법사** — 첫 번째 관리자 로그인 시 단계별 안내 설정. 활성화할 채널 선택 (음성, SMS, WhatsApp, Signal, 신고), 제공업체 설정, 핫라인 이름 지정.

**시작 체크리스트** — 설정 진행 상황을 추적하는 대시보드 위젯: 채널 설정, 자원봉사자 온보딩, 근무 생성.

**실시간 모니터링** — WebSocket을 통해 활성 통화, 대기 중인 발신자, 대화 및 자원봉사자 상태를 실시간으로 확인. 메트릭이 즉시 업데이트됩니다.

**자원봉사자 관리** — 생성된 키 쌍으로 자원봉사자를 추가하고, 역할 관리 (자원봉사자, 관리자, 신고자), 온라인 상태 확인. 역할 선택이 있는 자가 등록을 위한 초대 링크.

**감사 로깅** — 모든 응답된 통화, 생성된 메모, 보낸 메시지, 제출된 신고, 변경된 설정 및 관리자 작업이 기록됩니다. 관리자를 위한 페이지네이션 뷰어.

**통화 기록** — 날짜 범위, 전화번호 검색, 자원봉사자 배정 필터가 있는 검색 가능한 통화 기록. GDPR 준수 데이터 내보내기.

**인앱 도움말** — FAQ 섹션, 역할별 가이드, 키보드 단축키 및 보안에 대한 빠른 참조 카드. 사이드바와 명령 팔레트에서 접근 가능.

## 자원봉사자 경험

**명령 팔레트** — Ctrl+K (Mac에서는 Cmd+K)를 눌러 탐색, 검색, 빠른 메모 생성 및 테마 전환에 즉시 접근. 관리자 전용 명령은 역할에 따라 필터링됩니다.

**실시간 알림** — 수신 전화가 브라우저 벨소리, 푸시 알림 및 깜빡이는 탭 제목을 트리거합니다. 설정에서 각 알림 유형을 독립적으로 전환할 수 있습니다.

**자원봉사자 접속 상태** — 관리자는 실시간 온라인, 오프라인 및 휴식 중 인원을 확인합니다. 자원봉사자는 사이드바의 휴식 스위치를 전환하여 근무를 벗어나지 않고 수신 전화를 일시 중지할 수 있습니다.

**키보드 단축키** — ?를 눌러 사용 가능한 모든 단축키를 확인하세요. 페이지를 이동하고, 명령 팔레트를 열고, 마우스 없이 일반적인 작업을 수행하세요.

**메모 초안 자동 저장** — 메모가 브라우저에서 암호화된 초안으로 자동 저장됩니다. 페이지가 새로고침되거나 자원봉사자가 다른 곳으로 이동해도 작업이 보존됩니다. 초안은 로그아웃 시 localStorage에서 삭제됩니다.

**암호화된 데이터 내보내기** — 자원봉사자 자신의 키를 사용하여 GDPR 준수 암호화 파일(.enc)로 메모를 내보냅니다. 원래 작성자만 내보내기를 복호화할 수 있습니다.

**다크/라이트 테마** — 다크 모드, 라이트 모드 또는 시스템 테마를 따르는 것 사이에 전환합니다. 세션별로 선호도가 유지됩니다.

## 다국어 및 모바일

**12개 이상 언어** — 전체 UI 번역: 영어, 스페인어, 중국어, 타갈로그어, 베트남어, 아랍어, 프랑스어, 아이티 크리올어, 한국어, 러시아어, 힌디어, 포르투갈어 및 독일어. 아랍어 RTL 지원.

**프로그레시브 웹 앱** — 브라우저를 통해 모든 기기에 설치 가능. 서비스 워커가 오프라인 실행을 위해 앱 셸을 캐시합니다. 수신 전화에 대한 푸시 알림.

**모바일 우선 디자인** — 전화기와 태블릿을 위한 반응형 레이아웃. 접을 수 있는 사이드바, 터치 친화적 컨트롤 및 적응형 레이아웃.

## 인증 및 키 관리

**PIN 보호 로컬 키 저장소** — 비밀 키가 6자리 PIN으로 PBKDF2 (600,000회 반복) + XChaCha20-Poly1305를 사용하여 암호화됩니다. 원시 키는 sessionStorage나 브라우저 API에 절대 접촉하지 않습니다 — 메모리 내 클로저에만 존재하며 잠금 시 제로화됩니다.

**자동 잠금** — 키 관리자는 유휴 시간 초과 후 또는 브라우저 탭이 숨겨질 때 자동으로 잠깁니다. PIN을 다시 입력하여 잠금을 해제하세요. 유휴 시간 설정 가능.

**디바이스 연결** — 비밀 키를 노출하지 않고 새 디바이스를 설정하세요. QR 코드를 스캔하거나 짧은 프로비저닝 코드를 입력하세요. 임시 ECDH 키 교환을 사용하여 디바이스 간 암호화된 키를 안전하게 전송합니다. 프로비저닝 룸은 5분 후 만료됩니다.

**복구 키** — 온보딩 중에 Base32 형식의 복구 키(128비트 엔트로피)를 받습니다. 이것은 이전의 nsec 표시 흐름을 대체합니다. 계속하기 전에 암호화된 백업 다운로드가 필수입니다.

**개별 메모 전방 비밀성** — 각 메모는 고유한 랜덤 키로 암호화된 후, 각 인가된 독자를 위해 ECIES로 래핑됩니다. 신원 키가 유출되더라도 과거 메모가 노출되지 않습니다.

**Nostr 키 쌍 인증** — 자원봉사자는 Nostr 호환 키 쌍(nsec/npub)으로 인증합니다. BIP-340 Schnorr 서명 검증. 비밀번호 없음, 이메일 주소 없음.

**WebAuthn passkey** — 다중 디바이스 로그인을 위한 선택적 passkey 지원. 하드웨어 키 또는 생체 인식을 등록한 다음, 비밀 키를 입력하지 않고 로그인합니다.

**세션 관리** — 2단계 접근 모델: "인증되었지만 잠긴 상태" (세션 토큰만) vs "인증되고 잠금 해제된 상태" (PIN 입력, 전체 암호화 접근). 유휴 시간 초과 경고가 있는 8시간 세션 토큰.
