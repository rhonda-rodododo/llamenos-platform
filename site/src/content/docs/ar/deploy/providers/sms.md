تعيد رسائل SMS في Llamenos استخدام بيانات اعتماد مزود الاتصالات الصوتية الموجود. لا حاجة لخدمة SMS منفصلة — إذا كنت قد كوّنت Twilio أو SignalWire أو Vonage أو Plivo للصوت، فإن SMS يعمل بنفس الحساب.

## المزودون المدعومون

| المزود | دعم SMS | ملاحظات |
|--------|---------|---------|
| **Twilio** | نعم | SMS ثنائي الاتجاه عبر Twilio Messaging API |
| **SignalWire** | نعم | متوافق مع Twilio API — نفس الواجهة |
| **Vonage** | نعم | SMS عبر Vonage REST API |
| **Plivo** | نعم | SMS عبر Plivo Message API |
| **Asterisk** | لا | Asterisk لا يدعم SMS أصلياً |

## 1. تفعيل SMS في إعدادات المسؤول

انتقل إلى **إعدادات المسؤول > قنوات المراسلة** (أو استخدم معالج الإعداد عند تسجيل الدخول الأول) وفعّل **SMS**.

كوّن إعدادات SMS:
- **رسالة الرد التلقائي** — رسالة ترحيب اختيارية تُرسل لجهات الاتصال لأول مرة
- **رد خارج ساعات العمل** — رسالة اختيارية تُرسل خارج ساعات المناوبة

## 2. تكوين الـ webhook

وجّه webhook SMS لمزود الاتصالات إلى Worker الخاص بك:

```
POST https://your-worker.your-domain.com/api/messaging/sms/webhook
```

### Twilio / SignalWire

1. اذهب إلى Twilio Console > Phone Numbers > Active Numbers
2. اختر رقم هاتفك
3. تحت **Messaging**، عيّن رابط الـ webhook لـ "A message comes in" إلى الرابط أعلاه
4. عيّن طريقة HTTP إلى **POST**

### Vonage

1. اذهب إلى Vonage API Dashboard > Applications
2. اختر تطبيقك
3. تحت **Messages**، عيّن Inbound URL إلى رابط الـ webhook أعلاه

### Plivo

1. اذهب إلى Plivo Console > Messaging > Applications
2. أنشئ أو عدّل تطبيق مراسلة
3. عيّن Message URL إلى رابط الـ webhook أعلاه
4. عيّن التطبيق لرقم هاتفك

## 3. الاختبار

أرسل SMS إلى رقم هاتف خط الطوارئ. يجب أن ترى المحادثة تظهر في علامة تبويب **المحادثات** في لوحة المسؤول.

## كيف يعمل

1. تصل رسالة SMS إلى مزودك، الذي يرسل webhook إلى Worker الخاص بك
2. يتحقق Worker من توقيع الـ webhook (HMAC خاص بالمزود)
3. تُحلل الرسالة وتُخزن في ConversationDO
4. يتم إخطار المتطوعين في المناوبة عبر أحداث مرحّل Nostr
5. يرد المتطوعون من علامة تبويب المحادثات — تُرسل الردود عبر SMS API لمزودك

## ملاحظات أمنية

- تعبر رسائل SMS شبكة الناقل كنص واضح — مزودك والناقلون يمكنهم قراءتها
- تُخزن الرسائل الواردة في ConversationDO بعد الوصول
- تُجزأ أرقام هواتف المرسلين قبل التخزين (الخصوصية)
- يتم التحقق من توقيعات الـ webhook لكل مزود (HMAC-SHA1 لـ Twilio، إلخ)
