Plivo هو مزود اتصالات سحابي اقتصادي مع واجهة برمجة تطبيقات مباشرة. يستخدم التحكم في المكالمات المستند إلى XML بشكل مشابه لـ TwiML، مما يجعل التكامل مع Llamenos سلساً.

## المتطلبات الأساسية

- [حساب Plivo](https://console.plivo.com/accounts/register/) (رصيد تجريبي متاح)
- نسخة Llamenos منشورة ومتاحة عبر عنوان URL عام

## 1. إنشاء حساب Plivo

سجّل في [console.plivo.com](https://console.plivo.com/accounts/register/). بعد التحقق، يمكنك إيجاد **Auth ID** و **Auth Token** في الصفحة الرئيسية للوحة التحكم.

## 2. شراء رقم هاتف

1. انتقل إلى **Phone Numbers** > **Buy Numbers** في لوحة تحكم Plivo
2. اختر بلدك وابحث عن أرقام بقدرة صوتية
3. اشترِ رقماً

## 3. إنشاء تطبيق XML

يستخدم Plivo "تطبيقات XML" لتوجيه المكالمات:

1. انتقل إلى **Voice** > **XML Applications**
2. انقر على **Add New Application**
3. كوّن:
   - **Application Name**: Llamenos Hotline
   - **Answer URL**: `https://your-worker-url.com/telephony/incoming` (POST)
   - **Hangup URL**: `https://your-worker-url.com/telephony/status` (POST)
4. احفظ التطبيق

## 4. ربط رقم الهاتف

1. انتقل إلى **Phone Numbers** > **Your Numbers**
2. انقر على رقم خط الطوارئ الخاص بك
3. ضمن **Voice**، اختر تطبيق XML الذي أنشأته في الخطوة 3
4. احفظ

## 5. التكوين في Llamenos

1. سجّل الدخول كمسؤول
2. انتقل إلى **الإعدادات** > **مزود خدمة الاتصالات**
3. اختر **Plivo** من القائمة المنسدلة
4. أدخل:
   - **Auth ID**: من لوحة تحكم Plivo
   - **Auth Token**: من لوحة تحكم Plivo
   - **Phone Number**: الرقم الذي اشتريته (بصيغة E.164)
5. انقر على **حفظ**

## 6. اختبار الإعداد

اتصل برقم خط الطوارئ. يجب أن تسمع قائمة اختيار اللغة ويتم توجيهك عبر تدفق المكالمة العادي.

## إعداد WebRTC (اختياري)

يستخدم Plivo WebRTC حزمة Browser SDK مع بيانات الاعتماد الحالية:

1. انتقل إلى **Voice** > **Endpoints** في لوحة تحكم Plivo
2. أنشئ نقطة نهاية جديدة (تعمل كهوية هاتف المتصفح)
3. في Llamenos، انتقل إلى **الإعدادات** > **مزود خدمة الاتصالات**
4. فعّل **الاتصال عبر WebRTC**
5. انقر على **حفظ**

يولّد المحول رموز HMAC محددة زمنياً من Auth ID و Auth Token للمصادقة الآمنة في المتصفح.

## ملاحظات خاصة بـ Plivo

- **XML مقابل TwiML**: يستخدم Plivo صيغة XML خاصة به للتحكم في المكالمات، وهي مشابهة لكنها غير مطابقة لـ TwiML. يولّد محول Llamenos XML الصحيح لـ Plivo تلقائياً.
- **Answer URL مقابل Hangup URL**: يفصل Plivo بين معالج المكالمة الأولي (Answer URL) ومعالج نهاية المكالمة (Hangup URL)، بخلاف Twilio الذي يستخدم إشعار حالة واحد.
- **حدود المعدل**: لدى Plivo حدود لمعدل API تختلف حسب مستوى الحساب. لخطوط الطوارئ ذات الحجم العالي، تواصل مع دعم Plivo لزيادة الحدود.

## استكشاف الأخطاء وإصلاحها

- **"Auth ID invalid"**: معرّف المصادقة ليس بريدك الإلكتروني. ابحث عنه في الصفحة الرئيسية للوحة تحكم Plivo.
- **المكالمات لا تُوجه**: تحقق أن رقم الهاتف مرتبط بتطبيق XML الصحيح.
- **أخطاء Answer URL**: يتوقع Plivo استجابات XML صالحة. تحقق من سجلات Worker للأخطاء في الاستجابة.
- **قيود المكالمات الصادرة**: الحسابات التجريبية لها قيود على المكالمات الصادرة. قم بالترقية للاستخدام الإنتاجي.
