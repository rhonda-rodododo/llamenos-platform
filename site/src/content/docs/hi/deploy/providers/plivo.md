Plivo एक बजट-अनुकूल क्लाउड टेलीफ़ोनी प्रदाता है जिसका API सीधा है। यह TwiML के समान XML-आधारित कॉल कंट्रोल का उपयोग करता है, जो Llamenos के साथ इंटीग्रेशन को सहज बनाता है।

## पूर्वापेक्षाएँ

- एक [Plivo अकाउंट](https://console.plivo.com/accounts/register/) (ट्रायल क्रेडिट उपलब्ध)
- आपका Llamenos इंस्टेंस तैनात और पब्लिक URL से एक्सेसिबल

## 1. Plivo अकाउंट बनाएँ

[console.plivo.com](https://console.plivo.com/accounts/register/) पर साइन अप करें। वेरिफ़िकेशन के बाद, आप डैशबोर्ड होम पेज पर अपना **Auth ID** और **Auth Token** पा सकते हैं।

## 2. फ़ोन नंबर खरीदें

1. Plivo Console में **Phone Numbers** > **Buy Numbers** पर जाएँ
2. अपना देश चुनें और वॉइस क्षमता वाले नंबर खोजें
3. एक नंबर खरीदें

## 3. XML Application बनाएँ

Plivo कॉल रूट करने के लिए "XML Applications" का उपयोग करता है:

1. **Voice** > **XML Applications** पर जाएँ
2. **Add New Application** पर क्लिक करें
3. कॉन्फ़िगर करें:
   - **Application Name**: Llamenos Hotline
   - **Answer URL**: `https://your-worker-url.com/telephony/incoming` (POST)
   - **Hangup URL**: `https://your-worker-url.com/telephony/status` (POST)
4. एप्लिकेशन सेव करें

## 4. फ़ोन नंबर लिंक करें

1. **Phone Numbers** > **Your Numbers** पर जाएँ
2. अपने हॉटलाइन नंबर पर क्लिक करें
3. **Voice** के तहत, चरण 3 में बनाया गया XML Application चुनें
4. सेव करें

## 5. Llamenos में कॉन्फ़िगर करें

1. एडमिन के रूप में लॉग इन करें
2. **सेटिंग्स** > **टेलीफ़ोनी प्रदाता** पर जाएँ
3. प्रदाता ड्रॉपडाउन से **Plivo** चुनें
4. दर्ज करें:
   - **Auth ID**: Plivo Console डैशबोर्ड से
   - **Auth Token**: Plivo Console डैशबोर्ड से
   - **Phone Number**: आपने जो नंबर खरीदा (E.164 फ़ॉर्मेट)
5. **सेव** पर क्लिक करें

## 6. सेटअप टेस्ट करें

अपने हॉटलाइन नंबर पर कॉल करें। आपको भाषा चयन मेनू सुनाई देना चाहिए और सामान्य कॉल फ़्लो से गुज़ारा जाना चाहिए।

## WebRTC सेटअप (वैकल्पिक)

Plivo WebRTC आपके मौजूदा क्रेडेंशियल्स के साथ Browser SDK का उपयोग करता है:

1. Plivo Console में **Voice** > **Endpoints** पर जाएँ
2. एक नया endpoint बनाएँ (यह ब्राउज़र फ़ोन आइडेंटिटी के रूप में कार्य करता है)
3. Llamenos में, **सेटिंग्स** > **टेलीफ़ोनी प्रदाता** पर जाएँ
4. **WebRTC कॉलिंग** टॉगल ऑन करें
5. **सेव** पर क्लिक करें

एडाप्टर सुरक्षित ब्राउज़र प्रमाणीकरण के लिए आपके Auth ID और Auth Token से समय-सीमित HMAC टोकन जनरेट करता है।

## Plivo-विशिष्ट नोट्स

- **XML बनाम TwiML**: Plivo कॉल कंट्रोल के लिए अपना स्वयं का XML फ़ॉर्मेट उपयोग करता है, जो TwiML के समान है लेकिन समान नहीं है। Llamenos एडाप्टर स्वचालित रूप से सही Plivo XML जनरेट करता है।
- **Answer URL बनाम Hangup URL**: Plivo प्रारंभिक कॉल हैंडलर (Answer URL) को कॉल एंड हैंडलर (Hangup URL) से अलग करता है, Twilio के विपरीत जो एकल स्टेटस कॉलबैक का उपयोग करता है।
- **रेट लिमिट**: Plivo में API रेट लिमिट हैं जो अकाउंट टियर के अनुसार भिन्न होती हैं। हाई-वॉल्यूम हॉटलाइन के लिए, लिमिट बढ़ाने के लिए Plivo सपोर्ट से संपर्क करें।

## समस्या निवारण

- **"Auth ID invalid"**: Auth ID आपका ईमेल पता नहीं है। इसे Plivo Console डैशबोर्ड होम पेज पर खोजें।
- **कॉल रूट नहीं हो रही**: वेरिफ़ाई करें कि फ़ोन नंबर सही XML Application से लिंक है।
- **Answer URL एरर**: Plivo वैलिड XML रिस्पॉन्स की अपेक्षा करता है। रिस्पॉन्स एरर के लिए अपने Worker लॉग्स जाँचें।
- **आउटबाउंड कॉल प्रतिबंध**: ट्रायल अकाउंट में आउटबाउंड कॉलिंग की सीमाएँ हैं। प्रोडक्शन उपयोग के लिए अपग्रेड करें।
