Vonage (पहले Nexmo) मज़बूत अंतर्राष्ट्रीय कवरेज और प्रतिस्पर्धी मूल्य निर्धारण प्रदान करता है। यह Twilio से भिन्न API मॉडल का उपयोग करता है — Vonage Applications आपके नंबर, webhooks, और क्रेडेंशियल्स को एक साथ ग्रुप करते हैं।

## पूर्वापेक्षाएँ

- एक [Vonage अकाउंट](https://dashboard.nexmo.com/sign-up) (फ्री क्रेडिट उपलब्ध)
- आपका Llamenos इंस्टेंस तैनात और पब्लिक URL से एक्सेसिबल

## 1. Vonage अकाउंट बनाएँ

[Vonage API डैशबोर्ड](https://dashboard.nexmo.com/sign-up) पर साइन अप करें। अपना अकाउंट वेरिफ़ाई करें और डैशबोर्ड होम पेज से अपना **API Key** और **API Secret** नोट करें।

## 2. फ़ोन नंबर खरीदें

1. Vonage डैशबोर्ड में **Numbers** > **Buy numbers** पर जाएँ
2. अपना देश चुनें और **Voice** क्षमता वाला नंबर चुनें
3. नंबर खरीदें

## 3. Vonage Application बनाएँ

Vonage कॉन्फ़िगरेशन को "Applications" में ग्रुप करता है:

1. **Applications** > **Create a new application** पर जाएँ
2. एक नाम दर्ज करें (जैसे, "Llamenos Hotline")
3. **Voice** के तहत, इसे टॉगल ऑन करें और सेट करें:
   - **Answer URL**: `https://your-worker-url.com/telephony/incoming` (POST)
   - **Event URL**: `https://your-worker-url.com/telephony/status` (POST)
4. **Generate new application** पर क्लिक करें
5. कन्फ़र्मेशन पेज पर दिखाया गया **Application ID** सेव करें
6. **प्राइवेट की** फ़ाइल डाउनलोड करें -- कॉन्फ़िगरेशन के लिए आपको इसकी सामग्री की आवश्यकता होगी

## 4. फ़ोन नंबर लिंक करें

1. **Numbers** > **Your numbers** पर जाएँ
2. अपने हॉटलाइन नंबर के बगल में गियर आइकन पर क्लिक करें
3. **Voice** के तहत, चरण 3 में बनाया गया Application चुनें
4. **Save** पर क्लिक करें

## 5. Llamenos में कॉन्फ़िगर करें

1. एडमिन के रूप में लॉग इन करें
2. **सेटिंग्स** > **टेलीफ़ोनी प्रदाता** पर जाएँ
3. प्रदाता ड्रॉपडाउन से **Vonage** चुनें
4. दर्ज करें:
   - **API Key**: Vonage डैशबोर्ड होम पेज से
   - **API Secret**: Vonage डैशबोर्ड होम पेज से
   - **Application ID**: चरण 3 से
   - **Phone Number**: आपने जो नंबर खरीदा (E.164 फ़ॉर्मेट)
5. **सेव** पर क्लिक करें

## 6. सेटअप टेस्ट करें

अपने हॉटलाइन नंबर पर कॉल करें। आपको भाषा चयन मेनू सुनाई देना चाहिए। वेरिफ़ाई करें कि कॉल शिफ्ट पर मौजूद वॉलंटियर्स को रूट होती हैं।

## WebRTC सेटअप (वैकल्पिक)

Vonage WebRTC आपके पहले से बनाए गए Application क्रेडेंशियल्स का उपयोग करता है:

1. Llamenos में, **सेटिंग्स** > **टेलीफ़ोनी प्रदाता** पर जाएँ
2. **WebRTC कॉलिंग** टॉगल ऑन करें
3. **प्राइवेट की** की सामग्री दर्ज करें (आपने जो फ़ाइल डाउनलोड की उसका पूरा PEM टेक्स्ट)
4. **सेव** पर क्लिक करें

Application ID पहले से कॉन्फ़िगर है। Vonage ब्राउज़र प्रमाणीकरण के लिए प्राइवेट की का उपयोग करके RS256 JWT जनरेट करता है।

## Vonage-विशिष्ट नोट्स

- **NCCO बनाम TwiML**: Vonage XML मार्कअप के बजाय JSON फ़ॉर्मेट में NCCO (Nexmo Call Control Objects) का उपयोग करता है। Llamenos एडाप्टर स्वचालित रूप से सही फ़ॉर्मेट जनरेट करता है।
- **Answer URL फ़ॉर्मेट**: Vonage अपेक्षा करता है कि answer URL XML नहीं बल्कि JSON (NCCO) लौटाए। यह एडाप्टर द्वारा संभाला जाता है।
- **Event URL**: Vonage कॉल इवेंट्स (रिंगिंग, उत्तर दिया, पूरा हुआ) को event URL पर JSON POST रिक्वेस्ट के रूप में भेजता है।
- **प्राइवेट की सुरक्षा**: प्राइवेट की एन्क्रिप्टेड रूप में स्टोर की जाती है। यह कभी सर्वर से बाहर नहीं जाती -- इसका उपयोग केवल अल्पकालिक JWT टोकन जनरेट करने के लिए किया जाता है।

## समस्या निवारण

- **"Application not found"**: वेरिफ़ाई करें कि Application ID बिल्कुल मेल खाता है। आप इसे Vonage डैशबोर्ड में **Applications** के तहत पा सकते हैं।
- **कोई इनकमिंग कॉल नहीं**: सुनिश्चित करें कि फ़ोन नंबर सही Application से लिंक है (चरण 4)।
- **प्राइवेट की एरर**: `-----BEGIN PRIVATE KEY-----` और `-----END PRIVATE KEY-----` लाइनों सहित पूरा PEM कंटेंट पेस्ट करें।
- **अंतर्राष्ट्रीय नंबर फ़ॉर्मेटिंग**: Vonage को E.164 फ़ॉर्मेट चाहिए। `+` और कंट्री कोड शामिल करें।
