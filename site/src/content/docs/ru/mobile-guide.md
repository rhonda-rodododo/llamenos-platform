Мобильное приложение Llamenos позволяет волонтёрам принимать звонки, отвечать на сообщения и писать зашифрованные заметки с телефона. Оно создано на React Native и использует то же криптографическое ядро на Rust, что и настольное приложение.

## Что такое мобильное приложение?

Мобильное приложение является дополнением к настольному приложению. Оно подключается к тому же бэкенду Llamenos (Cloudflare Workers или самостоятельно размещённому) и использует тот же протокол, поэтому волонтёры могут беспрепятственно переключаться между настольным и мобильным вариантами.

Мобильное приложение находится в отдельном репозитории (`llamenos-mobile`), но использует общие компоненты:

- **llamenos-core** — тот же Rust-крейт для всех криптографических операций, скомпилированный через UniFFI для iOS и Android
- **Протокол** — тот же формат передачи данных, эндпоинты API и схема шифрования
- **Бэкенд** — тот же Cloudflare Worker или самостоятельно размещённый сервер

## Загрузка и установка

### Android

Мобильное приложение в настоящее время распространяется в виде APK для боковой установки:

1. Загрузите последний файл `.apk` со страницы [GitHub Releases](https://github.com/rhonda-rodododo/llamenos-mobile/releases/latest)
2. На вашем устройстве Android перейдите в **Настройки > Безопасность** и включите **Установка из неизвестных источников** (или включите это для конкретного приложения при появлении запроса)
3. Откройте загруженный APK и нажмите **Установить**
4. После установки откройте Llamenos из меню приложений

Распространение через App Store и Play Store запланировано на будущий выпуск.

### iOS

Сборки для iOS доступны в виде бета-релизов TestFlight:

1. Установите [TestFlight](https://apps.apple.com/app/testflight/id899247664) из App Store
2. Попросите администратора прислать ссылку-приглашение в TestFlight
3. Откройте ссылку на вашем устройстве iOS для участия в бета-тестировании
4. Установите Llamenos из TestFlight

Распространение через App Store запланировано на будущий выпуск.

## Начальная настройка

Мобильное приложение настраивается путём привязки к существующей учётной записи на настольном компьютере. Это гарантирует, что один и тот же криптографический идентификатор используется на всех устройствах без передачи секретного ключа в открытом виде.

### Подготовка устройства (сканирование QR-кода)

1. Откройте настольное приложение Llamenos и перейдите в **Настройки > Устройства**
2. Нажмите **Привязать новое устройство** — это создаёт QR-код с одноразовым токеном подготовки
3. Откройте мобильное приложение Llamenos и нажмите **Привязать устройство**
4. Отсканируйте QR-код камерой телефона
5. Приложения выполняют эфемерный обмен ключами ECDH для безопасной передачи зашифрованного материала ключей
6. Установите PIN на мобильном приложении для защиты локального хранилища ключей
7. Мобильное приложение теперь привязано и готово к использованию

В процессе подготовки ваш nsec никогда не передаётся в открытом виде. Настольное приложение оборачивает материал ключей с помощью эфемерного общего секрета, а мобильное приложение разворачивает его локально.

### Ручная настройка (ввод nsec)

Если вы не можете отсканировать QR-код, вы можете ввести nsec напрямую:

1. Откройте мобильное приложение и нажмите **Ввести nsec вручную**
2. Вставьте ваш ключ `nsec1...`
3. Установите PIN для защиты локального хранилища
4. Приложение выводит ваш публичный ключ и регистрируется в бэкенде

Этот метод требует непосредственной работы с nsec, поэтому используйте его только если привязка устройства невозможна. Используйте менеджер паролей для вставки nsec, а не вводите его вручную.

## Сравнение функций

| Функция | Настольный | Мобильный |
|---|---|---|
| Приём входящих звонков | Да | Да |
| Написание зашифрованных заметок | Да | Да |
| Пользовательские поля заметок | Да | Да |
| Ответ на сообщения (SMS, WhatsApp, Signal) | Да | Да |
| Просмотр переговоров | Да | Да |
| Статус смены и перерывы | Да | Да |
| Клиентская транскрипция | Да (WASM Whisper) | Нет |
| Поиск по заметкам | Да | Да |
| Командная палитра | Да (Ctrl+K) | Нет |
| Горячие клавиши | Да | Нет |
| Настройки администратора | Да (полные) | Да (ограниченные) |
| Управление волонтёрами | Да | Только просмотр |
| Просмотр журналов аудита | Да | Да |
| Вызов через браузер WebRTC | Да | Нет (использует нативный телефон) |
| Push-уведомления | Уведомления ОС | Нативный push (FCM/APNS) |
| Автообновление | Tauri updater | App Store / TestFlight |
| Файловые вложения (отчёты) | Да | Да |

## Ограничения

- **Нет клиентской транскрипции** — Модель WASM Whisper требует значительных ресурсов памяти и CPU, что нецелесообразно на мобильных устройствах. Транскрипция звонков доступна только на настольном компьютере.
- **Сниженная производительность криптографии** — Хотя мобильное приложение использует то же криптографическое ядро Rust через UniFFI, операции могут выполняться медленнее на устройствах низкого класса по сравнению с производительностью нативного настольного компьютера.
- **Ограниченные функции администратора** — Некоторые административные операции (массовое управление волонтёрами, подробная настройка параметров) доступны только в настольном приложении. Мобильное приложение предоставляет представления только для чтения для большинства экранов администратора.
- **Нет вызовов WebRTC** — Мобильные волонтёры получают звонки на свой номер телефона через провайдера телефонии, а не через браузер. Вызовы WebRTC в приложении доступны только на настольном компьютере.
- **Аккумулятор и подключение** — Приложению необходимо постоянное соединение для получения обновлений в реальном времени. Фоновый режим может быть ограничен управлением питанием ОС. Держите приложение на переднем плане во время смен для надёжных уведомлений.

## Устранение неполадок мобильного приложения

### Подготовка не удаётся с ошибкой "Недопустимый QR-код"

- Убедитесь, что QR-код был создан недавно (токены подготовки истекают через 5 минут)
- Создайте новый QR-код в настольном приложении и попробуйте снова
- Убедитесь, что оба устройства подключены к интернету

### Не получаю push-уведомления

- Убедитесь, что уведомления для Llamenos включены в настройках вашего устройства
- На Android: перейдите в **Настройки > Приложения > Llamenos > Уведомления** и включите все каналы
- На iOS: перейдите в **Настройки > Уведомления > Llamenos** и включите **Разрешить уведомления**
- Убедитесь, что режим "Не беспокоить" выключен
- Убедитесь, что ваша смена активна и вы не на перерыве

### Приложение вылетает при запуске

- Убедитесь, что используете последнюю версию приложения
- Очистите кэш приложения: **Настройки > Приложения > Llamenos > Хранилище > Очистить кэш**
- Если проблема сохраняется, удалите и переустановите приложение (потребуется повторная привязка устройства)

### Не удаётся расшифровать старые заметки после переустановки

- Переустановка приложения удаляет локальный материал ключей
- Повторно привяжите устройство через QR-код из настольного приложения для восстановления доступа
- Заметки, зашифрованные до переустановки, будут доступны после повторной привязки устройства с тем же идентификатором

### Низкая производительность на старых устройствах

- Закройте другие приложения для освобождения памяти
- Отключите анимации в настройках приложения, если доступно
- Рассмотрите использование настольного приложения для ресурсоёмких операций, таких как массовый просмотр заметок
