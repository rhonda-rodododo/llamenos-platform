SignalWire — это экономичная альтернатива Twilio с совместимым API. Он использует LaML (язык разметки, совместимый с TwiML), поэтому миграция между Twilio и SignalWire проста.

## Предварительные требования

- [Учётная запись SignalWire](https://signalwire.com/signup) (бесплатная пробная версия доступна)
- Ваш экземпляр Llamenos развёрнут и доступен по публичному URL

## 1. Создайте учётную запись SignalWire

Зарегистрируйтесь на [signalwire.com/signup](https://signalwire.com/signup). При регистрации вы выберете имя **Space** (например, `myhotline`). URL вашего пространства будет `myhotline.signalwire.com`. Запишите это имя — оно понадобится при настройке.

## 2. Купите номер телефона

1. В панели управления SignalWire перейдите в **Phone Numbers**
2. Нажмите **Buy a Phone Number**
3. Найдите номер с возможностью голосовой связи
4. Купите номер

## 3. Получите учётные данные

1. Перейдите в **API** в панели управления SignalWire
2. Найдите **Project ID** (выполняет функцию Account SID)
3. Создайте новый **API Token**, если у вас его нет — выполняет функцию Auth Token

## 4. Настройте вебхуки

1. Перейдите в **Phone Numbers** в панели управления
2. Нажмите на номер горячей линии
3. В разделе **Voice Settings** установите:
   - **Handle calls using**: LaML Webhooks
   - **When a call comes in**: `https://your-worker-url.com/telephony/incoming` (POST)
   - **Call status callback**: `https://your-worker-url.com/telephony/status` (POST)

## 5. Настройте в Llamenos

1. Войдите как администратор
2. Перейдите в **Настройки** > **Провайдер телефонии**
3. Выберите **SignalWire** из выпадающего списка
4. Введите:
   - **Account SID**: ваш Project ID из шага 3
   - **Auth Token**: ваш API Token из шага 3
   - **SignalWire Space**: имя вашего пространства (только имя, не полный URL — например, `myhotline`)
   - **Phone Number**: купленный номер (формат E.164)
5. Нажмите **Сохранить**

## 6. Проверьте настройку

Позвоните на номер горячей линии. Вы должны услышать меню выбора языка, за которым следует процесс обработки звонка.

## Настройка WebRTC (опционально)

SignalWire WebRTC использует тот же шаблон API-ключей, что и Twilio:

1. В панели управления SignalWire создайте **API Key** в разделе **API** > **Tokens**
2. Создайте **приложение LaML**:
   - Перейдите в **LaML** > **LaML Applications**
   - Установите Voice URL на `https://your-worker-url.com/telephony/webrtc-incoming`
   - Запишите Application SID
3. В Llamenos перейдите в **Настройки** > **Провайдер телефонии**
4. Включите **Звонки через WebRTC**
5. Введите API Key SID, API Key Secret и Application SID
6. Нажмите **Сохранить**

## Отличия от Twilio

- **LaML vs TwiML**: SignalWire использует LaML, который функционально идентичен TwiML. Llamenos обрабатывает это автоматически.
- **URL пространства**: API-вызовы направляются на `{space}.signalwire.com` вместо `api.twilio.com`. Адаптер обрабатывает это через указанное вами имя пространства.
- **Ценообразование**: SignalWire обычно на 30-40% дешевле Twilio для голосовых звонков.
- **Паритет функций**: Все функции Llamenos (запись, транскрипция, CAPTCHA, голосовая почта) работают идентично с SignalWire.

## Устранение неполадок

- **Ошибки «Space not found»**: Перепроверьте имя пространства (только поддомен, не полный URL).
- **Сбои вебхуков**: Убедитесь, что URL вашего Worker публично доступен и использует HTTPS.
- **Проблемы с API-токеном**: Токены SignalWire могут истекать. Создайте новый токен, если получаете ошибки аутентификации.
