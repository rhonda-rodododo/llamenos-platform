Plivo — бюджетный облачный провайдер телефонии с простым API. Он использует управление звонками на основе XML, аналогичное TwiML, что делает интеграцию с Llamenos бесшовной.

## Предварительные требования

- [Учётная запись Plivo](https://console.plivo.com/accounts/register/) (пробный кредит доступен)
- Ваш экземпляр Llamenos развёрнут и доступен по публичному URL

## 1. Создайте учётную запись Plivo

Зарегистрируйтесь на [console.plivo.com](https://console.plivo.com/accounts/register/). После верификации вы найдёте **Auth ID** и **Auth Token** на главной странице панели управления.

## 2. Купите номер телефона

1. Перейдите в **Phone Numbers** > **Buy Numbers** в консоли Plivo
2. Выберите страну и найдите номера с возможностью голосовой связи
3. Купите номер

## 3. Создайте XML-приложение

Plivo использует «XML-приложения» для маршрутизации звонков:

1. Перейдите в **Voice** > **XML Applications**
2. Нажмите **Add New Application**
3. Настройте:
   - **Application Name**: Llamenos Hotline
   - **Answer URL**: `https://your-worker-url.com/telephony/incoming` (POST)
   - **Hangup URL**: `https://your-worker-url.com/telephony/status` (POST)
4. Сохраните приложение

## 4. Привяжите номер телефона

1. Перейдите в **Phone Numbers** > **Your Numbers**
2. Нажмите на номер горячей линии
3. В разделе **Voice** выберите XML-приложение, созданное на шаге 3
4. Сохраните

## 5. Настройте в Llamenos

1. Войдите как администратор
2. Перейдите в **Настройки** > **Провайдер телефонии**
3. Выберите **Plivo** из выпадающего списка
4. Введите:
   - **Auth ID**: из панели управления Plivo Console
   - **Auth Token**: из панели управления Plivo Console
   - **Phone Number**: купленный номер (формат E.164)
5. Нажмите **Сохранить**

## 6. Проверьте настройку

Позвоните на номер горячей линии. Вы должны услышать меню выбора языка и пройти через стандартный процесс обработки звонка.

## Настройка WebRTC (опционально)

Plivo WebRTC использует Browser SDK с вашими существующими учётными данными:

1. Перейдите в **Voice** > **Endpoints** в консоли Plivo
2. Создайте новую конечную точку (выступает в качестве идентификатора телефона в браузере)
3. В Llamenos перейдите в **Настройки** > **Провайдер телефонии**
4. Включите **Звонки через WebRTC**
5. Нажмите **Сохранить**

Адаптер генерирует ограниченные по времени HMAC-токены из вашего Auth ID и Auth Token для безопасной аутентификации в браузере.

## Особенности Plivo

- **XML vs TwiML**: Plivo использует собственный XML-формат для управления звонками, который похож, но не идентичен TwiML. Адаптер Llamenos автоматически генерирует правильный XML для Plivo.
- **Answer URL vs Hangup URL**: Plivo разделяет начальный обработчик звонка (Answer URL) и обработчик завершения звонка (Hangup URL), в отличие от Twilio, который использует единый обратный вызов статуса.
- **Ограничения частоты**: Plivo имеет ограничения частоты API, которые зависят от уровня учётной записи. Для горячих линий с высокой нагрузкой обратитесь в службу поддержки Plivo для увеличения лимитов.

## Устранение неполадок

- **«Auth ID invalid»**: Auth ID — это не ваш адрес электронной почты. Найдите его на главной странице панели управления Plivo Console.
- **Звонки не маршрутизируются**: Убедитесь, что номер телефона привязан к правильному XML-приложению.
- **Ошибки Answer URL**: Plivo ожидает корректные XML-ответы. Проверьте логи вашего Worker на наличие ошибок ответа.
- **Ограничения исходящих звонков**: Пробные учётные записи имеют ограничения на исходящие звонки. Обновите для продакшен-использования.
