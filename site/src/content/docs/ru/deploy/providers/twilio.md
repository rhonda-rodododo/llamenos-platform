title: "Настройка: Twilio"
description: Пошаговое руководство по настройке Twilio в качестве провайдера телефонии.
---

Twilio — это провайдер телефонии по умолчанию для Llamenos и самый простой для начала работы. Это руководство описывает создание учётной записи, настройку номера телефона и конфигурацию вебхуков.

## Предварительные требования

- [Учётная запись Twilio](https://www.twilio.com/try-twilio) (бесплатная пробная версия подходит для тестирования)
- Ваш экземпляр Llamenos развёрнут и доступен по публичному URL

## 1. Создайте учётную запись Twilio

Зарегистрируйтесь на [twilio.com/try-twilio](https://www.twilio.com/try-twilio). Подтвердите электронную почту и номер телефона. Twilio предоставляет пробный кредит для тестирования.

## 2. Купите номер телефона

1. Перейдите в **Phone Numbers** > **Manage** > **Buy a number** в консоли Twilio
2. Найдите номер с возможностью **Voice** в нужном коде региона
3. Нажмите **Buy** и подтвердите

Сохраните этот номер — вы введёте его в настройках администратора Llamenos.

## 3. Получите Account SID и Auth Token

1. Перейдите на [панель управления Twilio Console](https://console.twilio.com)
2. Найдите **Account SID** и **Auth Token** на главной странице
3. Нажмите на значок глаза, чтобы показать Auth Token

## 4. Настройте вебхуки

В консоли Twilio перейдите к настройке вашего номера телефона:

1. Перейдите в **Phone Numbers** > **Manage** > **Active Numbers**
2. Нажмите на ваш номер горячей линии
3. В разделе **Voice Configuration** установите:
   - **A call comes in**: Webhook, `https://your-worker-url.com/telephony/incoming`, HTTP POST
   - **Call status changes**: `https://your-worker-url.com/telephony/status`, HTTP POST

Замените `your-worker-url.com` на фактический URL вашего Cloudflare Worker.

## 5. Настройте в Llamenos

1. Войдите как администратор
2. Перейдите в **Настройки** > **Провайдер телефонии**
3. Выберите **Twilio** из выпадающего списка провайдеров
4. Введите:
   - **Account SID**: из шага 3
   - **Auth Token**: из шага 3
   - **Phone Number**: купленный вами номер (формат E.164, например, `+15551234567`)
5. Нажмите **Сохранить**

## 6. Проверьте настройку

Позвоните на номер горячей линии с телефона. Вы должны услышать меню выбора языка. Если у вас есть волонтёры на смене, звонок будет направлен к ним.

## Настройка WebRTC (опционально)

Чтобы волонтёры могли отвечать на звонки в браузере вместо телефона:

### Создайте API Key

1. Перейдите в **Account** > **API keys & tokens** в консоли Twilio
2. Нажмите **Create API Key**
3. Выберите тип ключа **Standard**
4. Сохраните **SID** и **Secret** — секрет показывается только один раз

### Создайте TwiML App

1. Перейдите в **Voice** > **Manage** > **TwiML Apps**
2. Нажмите **Create new TwiML App**
3. Установите **Voice Request URL** на `https://your-worker-url.com/telephony/webrtc-incoming`
4. Сохраните и запишите **App SID**

### Включите в Llamenos

1. Перейдите в **Настройки** > **Провайдер телефонии**
2. Включите **Звонки через WebRTC**
3. Введите:
   - **API Key SID**: из созданного вами API-ключа
   - **API Key Secret**: из созданного вами API-ключа
   - **TwiML App SID**: из созданного вами приложения TwiML
4. Нажмите **Сохранить**

Для настройки волонтёров и устранения неполадок смотрите [Звонки через браузер (WebRTC)](/docs/deploy/providers/webrtc).

## Устранение неполадок

- **Звонки не поступают**: Убедитесь, что URL вебхука правильный и Worker развёрнут. Проверьте журналы ошибок в консоли Twilio.
- **Ошибки «Invalid webhook»**: Убедитесь, что URL вебхука использует HTTPS и возвращает корректный TwiML.
- **Ограничения пробной учётной записи**: Пробные учётные записи могут звонить только на подтверждённые номера. Обновите до платной учётной записи для продакшена.
- **Ошибки проверки вебхуков**: Убедитесь, что Auth Token в Llamenos совпадает с токеном в консоли Twilio.
