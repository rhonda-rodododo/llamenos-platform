Llamenos поддерживает обмен сообщениями через Signal с помощью самостоятельно размещённого моста [signal-cli-rest-api](https://github.com/bbernhard/signal-cli-rest-api). Signal обеспечивает наилучшую защиту конфиденциальности среди всех каналов обмена сообщениями, что делает его идеальным для чувствительных сценариев кризисного реагирования.

## Предварительные требования

- Сервер Linux или виртуальная машина для моста (может быть тем же сервером, что и Asterisk, или отдельным)
- Docker, установленный на сервере моста
- Выделенный номер телефона для регистрации Signal
- Сетевой доступ от моста к вашему Cloudflare Worker

## Архитектура

![Signal Bridge Architecture](/diagrams/signal-bridge.svg)

Мост signal-cli работает на вашей инфраструктуре и перенаправляет сообщения в ваш Worker через HTTP-вебхуки. Это означает, что вы контролируете весь путь сообщения от Signal до вашего приложения.

## 1. Развёртывание моста signal-cli

Запустите контейнер Docker signal-cli-rest-api:

```bash
docker run -d \
  --name signal-cli \
  --restart unless-stopped \
  -p 8080:8080 \
  -v signal-cli-data:/home/.local/share/signal-cli \
  -e MODE=json-rpc \
  bbernhard/signal-cli-rest-api:latest
```

## 2. Регистрация номера телефона

Зарегистрируйте мост с выделенным номером телефона:

```bash
# Request a verification code via SMS
curl -X POST http://localhost:8080/v1/register/+1234567890

# Verify with the code you received
curl -X POST http://localhost:8080/v1/register/+1234567890/verify/123456
```

## 3. Настройка перенаправления вебхуков

Настройте мост для перенаправления входящих сообщений в ваш Worker:

```bash
curl -X PUT http://localhost:8080/v1/about \
  -H "Content-Type: application/json" \
  -d '{
    "webhook": {
      "url": "https://your-worker.your-domain.com/api/messaging/signal/webhook",
      "headers": {
        "Authorization": "Bearer your-webhook-secret"
      }
    }
  }'
```

## 4. Включение Signal в настройках администратора

Перейдите в **Настройки администратора > Каналы обмена сообщениями** (или воспользуйтесь мастером настройки) и включите **Signal**.

Введите следующее:
- **URL моста** — URL вашего моста signal-cli (например, `https://signal-bridge.example.com:8080`)
- **API-ключ моста** — токен-носитель для аутентификации запросов к мосту
- **Секрет вебхука** — секрет для проверки входящих вебхуков (должен совпадать с тем, что вы настроили на шаге 3)
- **Зарегистрированный номер** — номер телефона, зарегистрированный в Signal

## 5. Тестирование

Отправьте сообщение Signal на ваш зарегистрированный номер телефона. Переговоры должны появиться на вкладке **Переговоры**.

## Мониторинг работоспособности

Llamenos отслеживает работоспособность моста signal-cli:
- Периодические проверки состояния эндпоинта моста `/v1/about`
- Плавное снижение функциональности при недоступности моста — другие каналы продолжают работать
- Оповещения администраторов при сбое моста

## Транскрипция голосовых сообщений

Голосовые сообщения Signal можно транскрибировать непосредственно в браузере волонтёра с помощью клиентского Whisper (WASM через `@huggingface/transformers`). Аудио никогда не покидает устройство — транскрипт шифруется и хранится вместе с голосовым сообщением в представлении переговоров. Волонтёры могут включать или выключать транскрипцию в своих личных настройках.

## Примечания по безопасности

- Signal обеспечивает сквозное шифрование между пользователем и мостом signal-cli
- Мост расшифровывает сообщения для их передачи в виде вебхуков — сервер моста имеет доступ к открытому тексту
- Аутентификация вебхуков использует токены-носители с постоянным сравнением по времени
- Держите мост в той же сети, что и ваш сервер Asterisk (если применимо), для минимальной уязвимости
- Мост хранит историю сообщений локально в своём томе Docker — рассмотрите шифрование в покое
- Для максимальной конфиденциальности: самостоятельно размещайте и Asterisk (голос), и signal-cli (обмен сообщениями) на вашей инфраструктуре

## Устранение неполадок

- **Мост не получает сообщения**: Убедитесь, что номер телефона правильно зарегистрирован с помощью `GET /v1/about`
- **Сбои доставки вебхуков**: Убедитесь, что URL вебхука доступен с сервера моста и заголовок авторизации совпадает
- **Проблемы регистрации**: Некоторые номера телефонов могут потребоваться сначала отвязать от существующей учётной записи Signal
