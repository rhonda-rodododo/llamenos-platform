---
title: "Настройка: SMS"
description: Включение входящих и исходящих SMS-сообщений через вашего провайдера телефонии.
---

SMS-сообщения в Llamenos используют существующие учётные данные вашего провайдера голосовой телефонии. Отдельный SMS-сервис не требуется — если вы уже настроили Twilio, SignalWire, Vonage или Plivo для голосовой связи, SMS работает с той же учётной записью.

## Поддерживаемые провайдеры

| Провайдер | Поддержка SMS | Примечания |
|----------|------------|-------|
| **Twilio** | Да | Полноценный двусторонний SMS через Twilio Messaging API |
| **SignalWire** | Да | Совместим с Twilio API — тот же интерфейс |
| **Vonage** | Да | SMS через Vonage REST API |
| **Plivo** | Да | SMS через Plivo Message API |
| **Asterisk** | Нет | Asterisk не поддерживает нативный SMS |

## 1. Включение SMS в настройках администратора

Перейдите в **Настройки администратора > Каналы обмена сообщениями** (или воспользуйтесь мастером настройки при первом входе) и включите **SMS**.

Настройте параметры SMS:
- **Автоответ** — необязательное приветственное сообщение, отправляемое новым контактам
- **Ответ в нерабочее время** — необязательное сообщение, отправляемое за пределами рабочих часов смены

## 2. Настройка вебхука

Укажите вебхук SMS вашего провайдера телефонии на ваш Worker:

```
POST https://your-worker.your-domain.com/api/messaging/sms/webhook
```

### Twilio / SignalWire

1. Перейдите в консоль Twilio > Phone Numbers > Active Numbers
2. Выберите ваш номер телефона
3. В разделе **Messaging** установите URL вебхука для "A message comes in" на указанный URL выше
4. Установите метод HTTP на **POST**

### Vonage

1. Перейдите на панель управления Vonage API > Applications
2. Выберите ваше приложение
3. В разделе **Messages** установите Inbound URL на указанный URL вебхука выше

### Plivo

1. Перейдите в консоль Plivo > Messaging > Applications
2. Создайте или отредактируйте приложение для обмена сообщениями
3. Установите Message URL на указанный URL вебхука выше
4. Привяжите приложение к вашему номеру телефона

## 3. Тестирование

Отправьте SMS на номер телефона вашей горячей линии. Переговоры должны появиться на вкладке **Переговоры** в панели администратора.

## Как это работает

1. SMS поступает к вашему провайдеру, который отправляет вебхук в ваш Worker
2. Worker проверяет подпись вебхука (специфичный для провайдера HMAC)
3. Сообщение разбирается и сохраняется в ConversationDO
4. Волонтёры, находящиеся на смене, уведомляются через события Nostr relay
5. Волонтёры отвечают с вкладки «Переговоры» — ответы отправляются обратно через SMS API вашего провайдера

## Примечания по безопасности

- SMS-сообщения проходят через сеть оператора в открытом виде — ваш провайдер и операторы могут их читать
- Входящие сообщения хранятся в ConversationDO после получения
- Номера телефонов отправителей хешируются перед хранением (конфиденциальность)
- Подписи вебхуков проверяются для каждого провайдера (HMAC-SHA1 для Twilio и т. д.)
