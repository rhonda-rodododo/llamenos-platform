---

WebRTC (Web Real-Time Communication) позволяет волонтёрам отвечать на звонки горячей линии прямо в браузере, без необходимости в телефоне. Это полезно для волонтёров, которые предпочитают не делиться своим номером телефона или работают с компьютера.

## Как это работает

1. Администратор включает WebRTC в настройках провайдера телефонии
2. Волонтёры устанавливают предпочтение звонков на «Браузер» в своём профиле
3. При поступлении звонка приложение Llamenos звонит в браузере с уведомлением
4. Волонтёр нажимает «Ответить», и звонок подключается через браузер с использованием микрофона

Аудио звонка маршрутизируется от провайдера телефонии через WebRTC-соединение в браузер волонтёра. Качество звонка зависит от интернет-соединения волонтёра.

## Предварительные требования

### Настройка администратора

- Поддерживаемый провайдер телефонии с включённым WebRTC (Twilio, SignalWire, Vonage или Plivo)
- Настроенные учётные данные WebRTC для провайдера (см. руководства по настройке провайдеров)
- WebRTC включён в **Настройки** > **Провайдер телефонии**

### Требования для волонтёров

- Современный браузер (Chrome, Firefox, Edge или Safari 14.1+)
- Работающий микрофон
- Стабильное интернет-соединение (минимум 100 кбит/с вверх/вниз)
- Предоставленные разрешения на уведомления браузера

## Настройка для конкретных провайдеров

Каждый провайдер телефонии требует разные учётные данные для WebRTC:

### Twilio / SignalWire

1. Создайте **API Key** в консоли провайдера
2. Создайте **приложение TwiML/LaML** с Voice URL, установленным на `https://your-worker-url.com/telephony/webrtc-incoming`
3. В Llamenos введите API Key SID, API Key Secret и Application SID

### Vonage

1. Ваше приложение Vonage уже включает возможности WebRTC
2. В Llamenos вставьте **закрытый ключ** вашего приложения (формат PEM)
3. Application ID уже настроен при первоначальной установке

### Plivo

1. Создайте **Endpoint** в консоли Plivo в разделе **Voice** > **Endpoints**
2. WebRTC использует ваши существующие Auth ID и Auth Token
3. Включите WebRTC в Llamenos — дополнительные учётные данные не нужны

### Asterisk

Asterisk WebRTC требует настройки SIP.js с транспортом WebSocket. Это сложнее, чем у облачных провайдеров:

1. Включите транспорт WebSocket в `http.conf` Asterisk
2. Создайте конечные точки PJSIP для WebRTC-клиентов с DTLS-SRTP
3. Llamenos автоматически настраивает клиент SIP.js при выборе Asterisk

Полные подробности в [руководстве по настройке Asterisk](/docs/deploy/providers/asterisk).

## Настройка предпочтений звонков волонтёра

Волонтёры настраивают предпочтения звонков в приложении:

1. Войдите в Llamenos
2. Перейдите в **Настройки** (значок шестерёнки)
3. В разделе **Предпочтения звонков** выберите **Браузер** вместо **Телефон**
4. Предоставьте разрешения на микрофон и уведомления при запросе
5. Держите вкладку Llamenos открытой во время смены

При поступлении звонка вы увидите уведомление браузера и индикатор звонка в приложении. Нажмите **Ответить**, чтобы подключиться.

## Совместимость браузеров

| Браузер | Десктоп | Мобильный | Примечания |
|---|---|---|---|
| Chrome | Да | Да | Рекомендуется |
| Firefox | Да | Да | Полная поддержка |
| Edge | Да | Да | На базе Chromium, полная поддержка |
| Safari | Да (14.1+) | Да (14.1+) | Требуется взаимодействие пользователя для запуска аудио |
| Brave | Да | Ограниченно | Может потребоваться отключить щиты для микрофона |

## Советы по качеству звука

- Используйте гарнитуру или наушники для предотвращения эха
- Закройте другие приложения, использующие микрофон
- По возможности используйте проводное интернет-соединение
- Отключите расширения браузера, которые могут мешать WebRTC (VPN-расширения, блокировщики рекламы с защитой от утечки WebRTC)

## Устранение неполадок

### Нет звука

- **Проверьте разрешения микрофона**: Нажмите на значок замка в адресной строке и убедитесь, что доступ к микрофону «Разрешён»
- **Протестируйте микрофон**: Используйте встроенный тест аудио в браузере или сайт вроде [webcamtest.com](https://webcamtest.com)
- **Проверьте аудиовыход**: Убедитесь, что ваши динамики или гарнитура выбраны в качестве устройства вывода

### Звонки не звонят в браузере

- **Уведомления заблокированы**: Проверьте, что уведомления браузера включены для сайта Llamenos
- **Вкладка не активна**: Вкладка Llamenos должна быть открыта (может быть в фоне, но должна существовать)
- **Предпочтение звонков**: Убедитесь, что предпочтение звонков установлено на «Браузер» в Настройках
- **WebRTC не настроен**: Попросите администратора проверить, что WebRTC включён и учётные данные установлены

### Проблемы с файрволом и NAT

WebRTC использует серверы STUN/TURN для прохождения через файрволы и NAT. Если звонки подключаются, но вы не слышите звука:

- **Корпоративные файрволы**: Некоторые файрволы блокируют UDP-трафик на нестандартных портах. Попросите IT-отдел разрешить UDP-трафик на портах 3478 и 10000-60000
- **Симметричный NAT**: Некоторые маршрутизаторы используют симметричный NAT, который может предотвратить прямые одноранговые соединения. TURN-серверы провайдера телефонии должны обработать это автоматически
- **Интерференция VPN**: VPN может мешать WebRTC-соединениям. Попробуйте отключить VPN во время смен

### Эхо или обратная связь

- Используйте наушники вместо динамиков
- Уменьшите чувствительность микрофона в настройках аудио ОС
- Включите подавление эха в браузере (обычно включено по умолчанию)
- Отодвиньтесь от твёрдых отражающих поверхностей
