Llamenos поддерживает обмен сообщениями WhatsApp Business через Meta Cloud API (Graph API v21.0). WhatsApp обеспечивает богатый обмен сообщениями с поддержкой текста, изображений, документов, аудио и интерактивных сообщений.

## Предварительные требования

- [Бизнес-аккаунт Meta](https://business.facebook.com)
- Номер телефона WhatsApp Business API
- Приложение разработчика Meta с включённым продуктом WhatsApp

## Режимы интеграции

Llamenos поддерживает два режима интеграции WhatsApp:

### Meta Direct (рекомендуется)

Подключитесь напрямую к Meta Cloud API. Обеспечивает полный контроль и все функции.

**Необходимые учётные данные:**
- **Phone Number ID** — идентификатор номера телефона вашего WhatsApp Business
- **Business Account ID** — идентификатор вашего бизнес-аккаунта Meta
- **Access Token** — долгосрочный токен доступа Meta API
- **Verify Token** — пользовательская строка для проверки вебхука
- **App Secret** — секрет вашего приложения Meta (для проверки подписи вебхука)

### Режим Twilio

Если вы уже используете Twilio для голосовой связи, вы можете маршрутизировать WhatsApp через вашу учётную запись Twilio. Более простая настройка, но некоторые функции могут быть ограничены.

**Необходимые учётные данные:**
- Ваш существующий Twilio Account SID, Auth Token и отправитель WhatsApp, подключённый к Twilio

## 1. Создание приложения Meta

1. Перейдите на [developers.facebook.com](https://developers.facebook.com)
2. Создайте новое приложение (тип: Business)
3. Добавьте продукт **WhatsApp**
4. В WhatsApp > Getting Started запишите ваши **Phone Number ID** и **Business Account ID**
5. Создайте постоянный токен доступа (Settings > Access Tokens)

## 2. Настройка вебхука

На панели разработчика Meta:

1. Перейдите в WhatsApp > Configuration > Webhook
2. Установите Callback URL на:
   ```
   https://your-worker.your-domain.com/api/messaging/whatsapp/webhook
   ```
3. Установите Verify Token на ту же строку, которую введёте в настройках администратора Llamenos
4. Подпишитесь на поле вебхука `messages`

Meta отправит GET-запрос для проверки вебхука. Ваш Worker ответит с челленджем, если токен проверки совпадает.

## 3. Включение WhatsApp в настройках администратора

Перейдите в **Настройки администратора > Каналы обмена сообщениями** (или воспользуйтесь мастером настройки) и включите **WhatsApp**.

Выберите режим **Meta Direct** или **Twilio** и введите необходимые учётные данные.

Настройте дополнительные параметры:
- **Автоответ** — отправляется новым контактам
- **Ответ в нерабочее время** — отправляется за пределами рабочих часов смены

## 4. Тестирование

Отправьте сообщение WhatsApp на номер телефона вашего Business. Переговоры должны появиться на вкладке **Переговоры**.

## 24-часовое окно сообщений

WhatsApp применяет 24-часовое окно для обмена сообщениями:
- Вы можете отвечать пользователю в течение 24 часов с момента его последнего сообщения
- По истечении 24 часов вы должны использовать одобренное **шаблонное сообщение** для возобновления переговоров
- Llamenos обрабатывает это автоматически — если окно истекло, отправляется шаблонное сообщение для возобновления переговоров

## Поддержка медиафайлов

WhatsApp поддерживает медиасообщения:
- **Изображения** (JPEG, PNG)
- **Документы** (PDF, Word и т. д.)
- **Аудио** (MP3, OGG)
- **Видео** (MP4)
- **Геолокация**
- **Интерактивные** кнопки и сообщения-списки

Медиавложения отображаются встроенными в представлении переговоров.

## Примечания по безопасности

- WhatsApp использует сквозное шифрование между пользователем и инфраструктурой Meta
- Meta технически может получить доступ к содержимому сообщений на своих серверах
- Сообщения хранятся в Llamenos после получения от вебхука
- Подписи вебхуков проверяются с использованием HMAC-SHA256 с секретом вашего приложения
- Для максимальной конфиденциальности рассмотрите использование Signal вместо WhatsApp
