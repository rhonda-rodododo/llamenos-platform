Llamenos 移动应用让志愿者可以通过手机接听来电、回复消息和撰写加密备注。它使用 React Native 构建，与桌面应用共享相同的 Rust 密码学核心。

## 什么是移动应用？

移动应用是桌面应用的伴侣。它连接到相同的 Llamenos 后端（Cloudflare Workers 或自托管），使用相同的协议，因此志愿者可以在桌面和移动端之间无缝切换。

移动应用位于单独的仓库（`llamenos-hotline`），但共享：

- **llamenos-core** —— 相同的 Rust crate 用于所有密码学操作，通过 UniFFI 编译为 iOS 和 Android
- **协议** —— 相同的通信格式、API 端点和加密方案
- **后端** —— 相同的 Cloudflare Worker 或自托管服务器

## 下载和安装

### Android

移动应用目前以 APK 形式分发，需要侧载安装：

1. 从 [GitHub Releases](https://github.com/rhonda-rodododo/llamenos-hotline/releases/latest) 页面下载最新的 `.apk` 文件
2. 在 Android 设备上，前往**设置 > 安全**，启用**允许安装未知来源**（或在提示时为特定应用启用）
3. 打开下载的 APK 并点击**安装**
4. 安装完成后，从应用列表中打开 Llamenos

App Store 和 Play Store 分发计划在未来版本中推出。

### iOS

iOS 版本通过 TestFlight 测试版发布：

1. 从 App Store 安装 [TestFlight](https://apps.apple.com/app/testflight/id899247664)
2. 向管理员索取 TestFlight 邀请链接
3. 在 iOS 设备上打开链接加入测试版
4. 从 TestFlight 安装 Llamenos

App Store 分发计划在未来版本中推出。

## 初始设置

移动应用通过与现有桌面账户关联来设置。这确保了在各设备间使用相同的密码学身份，而无需以明文传输密钥。

### 设备配对（扫描二维码）

1. 打开 Llamenos 桌面应用，前往**设置 > 设备**
2. 点击**关联新设备** —— 这会生成一个包含一次性配对令牌的二维码
3. 打开 Llamenos 移动应用，点击**关联设备**
4. 用手机摄像头扫描二维码
5. 应用程序执行临时 ECDH 密钥交换，安全地传输加密的密钥材料
6. 在移动应用上设置 PIN 以保护本地密钥存储
7. 移动应用现已关联，可以使用了

配对过程绝不会以明文传输您的 nsec。桌面应用使用临时共享密钥封装密钥材料，移动应用在本地解封。

### 手动设置（输入 nsec）

如果您无法扫描二维码，可以直接输入 nsec：

1. 打开移动应用，点击**手动输入 nsec**
2. 粘贴您的 `nsec1...` 密钥
3. 设置 PIN 以保护本地存储
4. 应用程序派生您的公钥并向后端注册

此方法需要直接处理您的 nsec，因此仅在设备关联不可行时使用。请使用密码管理器粘贴 nsec，而不是手动输入。

## 限制

- **无客户端语音转文字** —— WASM Whisper 模型需要大量内存和 CPU 资源，在移动端不切实际。通话转录仅在桌面端可用。
- **密码学性能较低** —— 虽然移动应用通过 UniFFI 使用相同的 Rust 密码学核心，但在低端设备上操作可能比桌面原生性能更慢。
- **管理功能有限** —— 某些管理操作（批量志愿者管理、详细设置配置）仅在桌面应用中可用。移动应用为大多数管理界面提供只读视图。
- **无 WebRTC 通话** —— 移动端志愿者通过电话服务提供商在其手机号码上接收来电，而不是通过浏览器。应用内 WebRTC 通话仅限桌面端。
- **电池和网络连接** —— 应用需要持续连接以接收实时更新。后台模式可能受操作系统电源管理限制。在值班期间请保持应用在前台以获得可靠的通知。

## 移动端故障排除

### 配对时出现"无效二维码"

- 确保二维码是最近生成的（配对令牌在 5 分钟后过期）
- 从桌面应用生成新的二维码并重试
- 确保两台设备都已连接到互联网

### 未收到推送通知

- 检查设备设置中是否为 Llamenos 启用了通知
- 在 Android 上：前往**设置 > 应用 > Llamenos > 通知**，启用所有频道
- 在 iOS 上：前往**设置 > 通知 > Llamenos**，启用**允许通知**
- 确保未处于勿扰模式
- 确认您的排班处于活动状态且未处于休息中

### 应用启动时崩溃

- 确保您正在运行最新版本的应用
- 清除应用缓存：**设置 > 应用 > Llamenos > 存储 > 清除缓存**
- 如果问题持续存在，请卸载并重新安装（需要重新关联设备）

### 重新安装后无法解密旧备注

- 重新安装应用会删除本地密钥材料
- 通过桌面应用的二维码重新关联设备以恢复访问
- 重新安装前加密的备注在使用相同身份重新关联设备后可以访问

### 旧设备上性能缓慢

- 关闭其他应用以释放内存
- 如果可用，在应用设置中禁用动画
- 对于批量备注查看等重度操作，建议使用桌面应用
